You are building a complete institutional website for **2A Educação**, a Brazilian edtech company. The design must be modern, tech-forward, youthful, and bold — inspired by the structure of napratica.org.br but fully reimagined with a premium, innovative identity.

---

## 🎨 BRAND IDENTITY

- **Company name:** 2A Educação
- **Color palette:** Black (#0A0A0A), White (#FFFFFF), and Yellow (#F5E642 or similar electric/acid yellow)
- **Tone:** Innovative, energetic, youthful, tech-forward, trustworthy
- **Typography:** Use expressive, modern fonts — such as Clash Display or Syne for headings, and Plus Jakarta Sans or DM Sans for body. Avoid Inter, Roboto, Arial.
- **Visual language:** Dark-dominant sections alternated with white sections, yellow used as a sharp accent/highlight color — not dominant, but electric. Think: a startup that takes careers seriously.

---

## ✨ DESIGN STYLE & MOTION

- **Aesthetic direction:** Premium dark-tech meets youthful energy. Think Apple x Notion x Y Combinator but Brazilian and bold.
- **Motion:** Implement smooth scroll-triggered animations (fade-in-up, stagger reveals, counter number animations). Add subtle particle effects or glowing orb backgrounds on hero section. Use yellow glow/shimmer accents on key CTA elements. Hover states should feel alive (scale, glow, color shift). Cards should have smooth hover elevation and shine effects.
- **Background effects:** Grain texture overlay on dark sections, gradient mesh or radial glow in yellow/amber on hero. Glassmorphism on cards where appropriate.
- **Micro-interactions:** Buttons with shimmer sweep animation on hover. Navigation links with underline slide. Section counters animate on scroll into view.
- **No stock/generic AI patterns.** Every design decision must feel intentional and specific to this brand.

## 3. Layout

### Container e Largura Máxima

O site usa um container centralizado com largura máxima de **1440px** em todas as seções:
```css
.container {
  width: 100%;
  max-width: 1440px;
  margin: 0 auto;
  padding: 0 16px; /* mobile */
}

/* Desktop (xl:) */
@media (min-width: 1280px) {
  .container {
    padding: 0 72px; /* xl:px-18 = 4.5rem = 72px */
  }
}
```

### Sistema de Grid

O layout principal usa **CSS Grid de 12 colunas** com `gap: 24px`:
```css
.grid-12 {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 24px;
}
```

Breakpoints de coluna usados:
- `col-span-12` — full width (mobile)
- `col-span-6` — metade da largura (desktop, hero)
- `col-span-4` — um terço (grid de 3 colunas)
- `grid-cols-1` → `md:grid-cols-4` — carrossel de vagas

### Padding e Margens Padrão

| Contexto               | Valor Tailwind       | Valor Computado  |
|------------------------|----------------------|------------------|
| Seção padrão           | `py-12`              | `48px` top/bottom|
| Hero section           | `py-6 lg:py-17`      | `24px / 68px`    |
| Form container         | `p-10`               | `40px`           |
| Card padrão            | `p-4`                | `16px`           |
| Card CTA verde         | `p-6`                | `24px`           |
| Botões                 | `px-6 py-[10px]`     | `24px / 10px`    |

### Breakpoints Responsivos

O site utiliza os breakpoints padrão do **Tailwind CSS**:

| Breakpoint | Prefixo  | Min-width |
|------------|----------|-----------|
| Mobile     | (nenhum) | `0px`     |
| Medium     | `md:`    | `768px`   |
| Large      | `lg:`    | `1024px`  |
| XL         | `xl:`    | `1280px`  |

Comportamentos responsivos notáveis:
- Navbar top (botões de ação) e navbar de navegação ficam em linhas separadas no desktop, colapsam para mobile
- A Hero passa de **1 coluna** (mobile) para **2 colunas de 6** (desktop)
- O grid de artigos vai de `grid-cols-1` para `md:grid-cols-4`
- Padding lateral do container: `px-4` (mobile) → `xl:px-18` (desktop)
